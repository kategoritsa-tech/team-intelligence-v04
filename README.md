# Team Intelligence

Инструмент для анализа daily-отчетов команды из Telegram HTML-экспорта.

Проект помогает PM / тимлиду быстро понять:

- кто регулярно пишет daily-отчеты;
- насколько отчеты полные и конкретные;
- где есть проблемы, блокеры и риски;
- кто опаздывает с отчетами;
- у кого системно слабые отчеты;
- какие антипаттерны в отчетности повторяются чаще всего.

Текущая стабильная версия: **v0.5.3**

---

## Что анализируется

На вход подается HTML-экспорт сообщений Telegram:

```text
data/messages.html
```

Система ищет daily-отчеты, парсит их структуру и считает метрики по отчетам и авторам.

Ожидаемая структура daily-отчета:

```text
Вчера:
- что было сделано

Сегодня:
- что планируется

Проблемы:
- блокеры / риски / нет проблем
```

Также поддерживаются варианты:

```text
вчера:
сегодня:
проблемы:
блокеры:
```

---

## Как запустить

Из корня проекта:

```bash
cd src
python3 -m team_intelligence --input ../data/messages.html
```

После запуска отчеты появятся в папке:

```text
src/reports/
```

## Excel-отчет

Excel — основной детальный рабочий отчет для анализа.

Файл:

```text
src/reports/team_intelligence.xlsx
```

### Листы Excel

#### Сводка

Краткая управленческая картина:

- всего отчетов;
- всего авторов;
- проблемные отчеты;
- среднее качество;
- средняя полнота;
- опоздания;
- авторы высокого и среднего риска;
- самый частый антипаттерн.

#### Проверка отчетов

Очередь отчетов, которые требуют внимания лида.

Туда попадают отчеты, где есть:

- низкое качество;
- низкая полнота;
- отсутствие выполненных задач;
- отсутствие плана;
- опоздание;
- проблемы или блокеры;
- замечания анализатора.

#### Риски по авторам

Агрегация проблем по каждому сотруднику.

Помогает понять:

- с кем нужно поговорить;
- кому нужна помощь;
- где есть системная проблема;
- кто регулярно пишет неполные или слабые отчеты.

#### Антипаттерны

Список повторяющихся проблем в daily-отчетах:

- нет выполненных задач;
- нет плана на сегодня;
- мало конкретики;
- есть проблемы или блокеры;
- отчет после дедлайна;
- нет привязки к задаче.

#### Все отчеты

Полная таблица daily-отчетов с метриками и исходным текстом отчета.

Используется для ручной проверки и разбора конкретных примеров.

#### Авторы по дням

Сводка активности авторов по дням.

## PDF-отчет

PDF — короткая управленческая выжимка для лида.

Файл:

```text
src/reports/team_intelligence.pdf
```

Содержит разделы:

1. Сводка
2. Ключевые риски
3. Проблемные отчеты
4. Риски по авторам
5. Частые антипаттерны
6. Выводы для лида

PDF не содержит все отчеты полностью. Для детального анализа используется Excel.

---

## Метрики качества

Система оценивает daily-отчеты по нескольким признакам:

- есть ли блок «Вчера»;
- есть ли блок «Сегодня»;
- есть ли блок «Проблемы» или «Блокеры»;
- указаны ли выполненные задачи;
- указан ли план;
- есть ли конкретика;
- есть ли привязка к задачам;
- есть ли опоздание по дедлайну;
- есть ли повторяющиеся проблемы.

Оценки не являются финальным вердиктом по сотруднику.  
Они помогают лиду быстрее найти отчеты, которые нужно проверить вручную.

## Важное правило по Git

Сгенерированные отчеты могут содержать реальные данные сотрудников.

Не коммитить реальные отчеты:

```text
src/reports/team_intelligence.xlsx
src/reports/team_intelligence.pdf
src/reports/daily_metrics.csv
src/reports/authors.csv
src/reports/report.txt
```

Если после запуска проекта `git status` показывает измененные отчеты, их нужно откатить:

```bash
git restore src/reports/team_intelligence.xlsx src/reports/team_intelligence.pdf src/reports/report.txt src/reports/authors.csv src/reports/daily_metrics.csv
```
